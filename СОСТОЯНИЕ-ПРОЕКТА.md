# BQDiptv — актуальное состояние проекта (снимок)

IPTV-плеер для Android TV и телефона. Kotlin + Media3/ExoPlayer + NanoHTTPD + ZXing.

## Ключевые факты (не менять)
- applicationId / namespace: **ru.bqd.iptv** (на нём держится автообновление «поверх»)
- Репозиторий обновлений: **https://github.com/DaretsE/BQD-ipTv**
- Подпись: постоянный keystore (alias bqdkey), 4 секрета в GitHub:
  KEYSTORE_BASE64, KEYSTORE_PASSWORD, KEY_ALIAS, KEY_PASSWORD
- Версия задаётся автоматически: versionCode = номер сборки Actions,
  versionName = 1.0.<номер>, тег релиза v<номер>
- Сборка: GitHub Actions (.github/workflows/build.yml) → Releases: BQDiptv.apk + version.json
- Минимум Android 5 (minSdk 21), target 34
- Приложение — только плеер; каналы добавляет пользователь (M3U + EPG XMLTV)

## Структура исходников (app/src/main/java/ru/bqd/iptv/)
- Models.kt — Channel, Playlist, PlaylistCfg, EpgSrc, M3uParser, CatchupHelper
- Store.kt — настройки (prefs "bqd_iptv"), плейлисты, EPG-источники, кэш; задаёт EpgManager.cacheFile
- Helpers.kt — Net (gzip), ImageLoader (LruCache+диск), EpgManager (двойная буферизация,
  ДИСКОВЫЙ КЭШ EPG, поиск с индексом и нестрогим совпадением)
- UpdateManager.kt — проверка/скачивание/установка обновлений, откат, installDownloaded,
  canInstall (продолжение установки после выдачи разрешения)
- WebConfigServer.kt — веб-настройка по локальной сети (порт 8765), управление плейлистами/EPG
- ChannelAdapter.kt / CategoryAdapter.kt / SearchAdapter.kt — списки
- PlayerActivity.kt — главный экран: плеер, меню, EPG-панель, браузер каналов, поиск,
  настройки, обновления, режим ТВ/Телефон, усыпление, скринсейвер

## Реализованные возможности
- EPG/иконки с двойной буферизацией + АТОМАРНЫЙ дисковый кэш (не пропадают после
  перезапуска/обновления/долгого простоя; при старте показываются из кэша мгновенно,
  сеть обновляет EPG в фоне и подменяет данные без «пустоты»)
- Принудительное обновление плейлиста и EPG (веб + настройки)
- Режим ТВ/Телефон с автоопределением; на телефоне экран первичной настройки
- Браузер каналов с превью и подписью «осталось …»
- Правая панель телепрограммы; прогресс текущей передачи; пометка передач, доступных
  в архиве (▶ · архив) с учётом окна catchup-days канала
- Управление несколькими плейлистами и EPG-источниками (имя, переименование, удаление)
- QR-код для страницы настройки на ТВ
- Адаптивное качество: Авто / Стабильность / Максимум
- Автообновление через GitHub Releases + установка поверх; авто-продолжение установки
  после выдачи разрешения; откат на предыдущую версию
- Поиск передач по названию и описанию (нестрогий: регистр/пробелы/частичное/опечатки),
  категории «сейчас / будущее / архив», переход на канал или в архив (catchup)
- Показ версии в заголовке «Настройки»
- Чёткая подсветка выбранного пункта на ТВ (selector list_focus)
- Без напоминаний о передачах (функционал убран)

## Известные открытые вопросы
- Предупреждение Google Play Protect при установке (нормально для приложений не из Google Play;
  полностью убирается только публикацией в магазин)
- Поиск зависит от качества сопоставления каналов с EPG (tvg-id/имена); при пустом
  результате интерфейс показывает диагностику (сколько каналов сопоставлено, сколько передач)

## Ревизия 2 (2026-06-16): исправления стабильности — что и почему
Изменены 4 файла: AndroidManifest.xml, Helpers.kt, Store.kt, PlayerActivity.kt.

1. Вылет при удалении/смене плейлиста (не запускался до сброса данных).
   Причина: большая EPG (url-tvg, напр. smolnp) → OutOfMemoryError (это Error, не Exception,
   старый catch(Exception) его не ловил) в фоне → краш процесса при каждом старте, т.к. cfg
   плейлиста уже сохранён. Правки: largeHeap=true (Manifest); все фоновые потоки EPG/поиска
   обёрнуты в catch(Throwable)+finally; лимиты MAX_PROGRAMS/MAX_PER_CHANNEL и обрезка
   title/desc при парсинге; Net.downloadText с лимитом размера (защита от гигантского m3u/xml);
   в Store — installCrashGuard(): перед падением удаляет epg_cache.bin(.tmp), чтобы следующий
   запуск гарантированно прошёл без ручного сброса.

2/3. Значки и EPG пропадали после обновления версии и после долгого простоя (полная 5-мин
   перезагрузка по сети; «нет данных» до принудительного обновления).
   Причина: дисковый кэш писался НЕ атомарно → мог остаться битым → чтение падало → пустой
   кэш → полная сетевая перезагрузка; плюс onCreate всегда форсил сетевой reload EPG.
   Правки: атомарная запись кэша (запись в .tmp + renameTo); чтение в catch(Throwable) с
   удалением битого файла; CACHE_FORMAT 2→3 (инвалидирует возможно-битые старые кэши —
   ОДИН раз EPG перезагрузится после этого обновления, дальше стабильно); старт из кэша
   мгновенно, сетевое обновление в фоне с подменой; при свежем кэше (<6 ч) сеть не трогается.

4. Поиск на ТВ не работал (2 буквы → «Поиск» и ничего; «лупа» не прятала клавиатуру).
   Причина: полноэкранный leanback-IME перекрывал результаты; построение индекса по всем
   title+desc → OOM. Правки: по actionSearch клавиатура принудительно скрывается и фокус
   переводится на список результатов; индекс хранит только нормализованный title, desc
   сканируется на лету и лишь при разумном объёме базы.

5. Метки архива в телепрограмме.
   Прошедшие передачи в окне архива помечаются «▶ · архив», текущая «● · эфир», будущие без
   метки; окно = catchupDays канала (по умолчанию 7 дн.). Клик по архивной запускает catchup;
   вне окна — подсказка «вне доступного архива (хранится N дн.)».

## Как выпустить новую версию
Залить изменённые файлы в репозиторий (Add file → Upload files) → push в main запускает
сборку → новая версия появляется в Releases → на ТВ Настройки → Проверить обновление.
versionCode/versionName и секреты трогать НЕ нужно.
