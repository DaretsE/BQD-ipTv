# Как залить это на GitHub и получить версию 2.0

Репозиторий: https://github.com/DaretsE/BQD-ipTv
Работа только через браузер, ничего компилировать локально не нужно.

## Что изменилось в этой поставке

Изменённые файлы (5):
- `app/build.gradle` — версия теперь 2.0
- `.github/workflows/build.yml` — версия 2.0 + автоскачивание шрифта иконок
- `app/src/main/java/ru/bqd/iptv/Store.kt` — новые настройки качества
- `app/src/main/java/ru/bqd/iptv/PlayerActivity.kt` — исправлен баг звука, подключены настройки качества, убран экран выбора режима и быстрое меню

Новые файлы (3):
- `app/src/main/java/ru/bqd/iptv/Quality.kt` — реальная логика качества трансляции
- `app/src/main/java/ru/bqd/iptv/GroupIcons.kt` — автоподбор иконок для групп каналов
- `app/src/main/java/ru/bqd/iptv/IconFont.kt` — иконочный шрифт интерфейса

Остальные файлы проекта в архиве лежат без изменений — можно залить всё целиком,
GitHub перезапишет только то, что реально отличается.

## Порядок заливки (5 минут)

1. Скачай и распакуй архив. Внутри — папка `BQDiptv` с деревом проекта.
2. Открой https://github.com/DaretsE/BQD-ipTv
3. Нажми **Add file → Upload files**.
4. Перетащи в окно загрузки содержимое папки `BQDiptv`: папки `app` и `.github`
   (и, если хочешь обновить описание, файлы `СОСТОЯНИЕ-ПРОЕКТА.md`
   и `КАК-ОБНОВИТЬ-НА-GITHUB.md`).
   Перетаскивать нужно **папками**, чтобы сохранились пути.
   Если браузер не даёт перетащить папку `.github` (она скрытая), обнови только
   `app`, а `build.yml` замени отдельно — см. пункт 6.
5. Внизу впиши описание коммита, например:
   `2.0: рабочие настройки качества, фикс звука при смене буфера, иконки групп`
   Выбери **Commit directly to the main branch** и нажми **Commit changes**.
6. Если `.github` перетащить не удалось: открой в репозитории файл
   `.github/workflows/build.yml`, нажми карандаш (Edit), выдели всё, вставь
   содержимое нового `build.yml` из архива и нажми Commit changes.

## Что произойдёт дальше

Push в `main` автоматически запустит сборку (вкладка **Actions**). Она:
- скачает шрифт иконок в `app/src/main/assets/icons.otf` (в репозиторий он не
  кладётся, качается при каждой сборке);
- соберёт и подпишет APK;
- опубликует релиз с файлами `BQDiptv.apk` и `version.json`.

Версия будет называться **2.0.<номер сборки>**, тег — `v<номер сборки>`.

На телевизоре: **Настройки → Проверить обновление**. Обновление подхватится,
потому что проверка сравнивает `versionCode` (номер сборки Actions), а он
продолжает расти как раньше. Секреты подписи и схему versionCode не трогаем.

## Если сборка упала

Открой Actions → последний запуск → красный шаг. Частые причины:
- шаг «Download icon font» — нет сети до raw.githubusercontent.com, просто
  перезапусти сборку кнопкой Re-run jobs;
- шаг «Build release APK» — ошибка компиляции: пришли мне текст ошибки,
  поправлю точечно.

## Новые настройки в приложении

Настройки → **Качество трансляции**:
- Потолок качества (авто / стабильность 720p / максимум) — меняется на лету;
- Размер буфера;
- **Задержка от эфира** — главный параметр для слабого интернета на даче:
  20–30 секунд отставания позволяют пережить провалы связи незаметно;
- **При обрыве связи** — «упорно» тихо переподключается вместо показа ошибки;
- **Тип декодера** — программный помогает, если картинка сыпется;
- **Автофреймрейт** — подстройка частоты экрана под поток (24/25/30/50/60).
